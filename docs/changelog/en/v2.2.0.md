# v2.2.0 Release Notes

**Release Date**: March 22, 2026  
**Status**: Stable  
**Upgrade Priority**: Recommended

---

## 🎯 Overview

Version 2.2.0 completes the documentation infrastructure with a fully navigable documentation site, enhances the CI/CD pipeline with integrated testing, and establishes comprehensive test coverage for the synchronization main link.

---

## ✨ New Features

### 📚 Documentation Site

- **Complete Navigation**: Added `changelog/index.md` as browsable archive
- **Deep Documentation Pages**: 
  - `architecture.md` - System design and module relationships
  - `deployment.md` - Production deployment guide
  - `security-sync.md` - Security boundaries and sync mechanisms
- **Front Matter Integration**: All documentation files now include Jekyll front matter and permalinks
- **GitHub Pages Optimization**: SEO metadata, excluded build artifacts, path triggers

### 🔧 CI Pipeline Enhancement

- **Split CI Jobs**: Separate `client` and `server` jobs for parallel execution
- **Test Integration**: Both test suites now run in CI pipeline
- **Concurrency Control**: Configured to prevent duplicate runs
- **Pages Deployment**: Added `actions/configure-pages@v5` step

### ✅ Test Coverage

#### Client Tests (`useSocket.test.js`)
- Connection establishment and teardown
- `sync-update` event handling
- `request-sync` flow
- Error handling scenarios

#### Server Tests (`index.test.js`)
- Health check endpoint
- `join-chain` event flow
- `push-update` processing
- `request-sync` responses

## 🐛 Bug Fixes

| Issue | Description | Fix |
|-------|-------------|-----|
| Timer Cleanup | Server timers not unref'd causing test hangs | Added `unref()` to all timers |
| Process Exit | `gracefulShutdown` and `startServer` calling `process.exit()` in tests | Skip `process.exit()` in test environment |
| Socket Testing | Unable to directly test socket events | Extracted `handleSocketConnection` for direct testing |

## 🔒 Security

No security changes in this release. Current security posture:

- ✅ End-to-end encryption (AES-256)
- ✅ Input validation on all endpoints
- ✅ Rate limiting (30 updates/min per socket)
- ✅ Size limits (5MB max payload)
